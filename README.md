# AWS Budgeting Automation

This repository contains Python scripts and Terraform configuration to create an AWS budget alert, configure IAM permissions for a budget automation user, and optionally stop running EC2 instances when a budget threshold is reached.

## Project Overview

- `budget_python.py` - creates an AWS budget with a monthly cost limit and email notification.
- `Budget-user.py` - creates an IAM user named `budget-user`, attaches a budget/SNS/EC2 permission policy, and generates access keys.
- `budget-user-permissions.py` - updated IAM policy script that creates or updates the `BudgetUserPolicy` and attaches it to `budget-user`.
- `main.tf` - Terraform configuration to create the same IAM user, policy, access key, and budget resources declaratively.
- `log.txt` - runtime log file generated by the Python scripts.

## Prerequisites

- Python 3.x
- `boto3` Python package
- AWS CLI configured with access to create IAM, Budgets, SNS, and EC2 resources
- Terraform installed if using `main.tf`

## Setup

1. Install Python dependencies:

   ```bash
   pip install boto3
   ```

2. Configure AWS credentials / profile, for example:

   ```bash
   aws configure --profile budget-user
   ```

3. Edit the Python scripts as needed:

   - `budget_python.py`:
     - `ACCOUNT_ID` should be your AWS account ID.
     - `BUDGET_NAME`, budget amount, and email address should be updated as needed.
   - `main.tf`:
     - update `region` and `address` under `aws_budgets_budget`.

## Usage

### Option 1: Python scripts

1. Create the budget automation IAM user and attach permissions:

   ```bash
   python Budget-user.py
   ```

   Or to create/update and attach the policy with versioning:

   ```bash
   python budget-user-permissions.py
   ```

2. Create the AWS budget and notification:

   ```bash
   python budget_python.py
   ```

3. If you want to test EC2 shutdown logic manually, uncomment or call `shutdown_instances()` in `budget_python.py`.

### Option 2: Terraform

1. Initialize Terraform:

   ```bash
   terraform init
   ```

2. Apply the Terraform configuration:

   ```bash
   terraform apply
   ```

   This will create the IAM user, policy, access key, and AWS budget resources.

## How it works

- `Budget-user.py` / `budget-user-permissions.py` manage IAM setup for a `budget-user` capable of creating and managing budgets, SNS topics/subscriptions, and stopping EC2 instances.
- `budget_python.py` creates a monthly cost budget of `$40` and configures an email notification at `100%` of the budget.
- `main.tf` provides the same infrastructure-as-code approach with Terraform.

## Notes

- The scripts write logs to `log.txt`.
- Access keys are sensitive and should be stored securely.
- The EC2 shutdown logic in `budget_python.py` is provided as a helper and is not executed automatically by the current `__main__` block.
- Update the email address in the budget notification to receive alerts.

## File reference

- `budget_python.py` - AWS budget creation + optional EC2 shutdown helper
- `Budget-user.py` - IAM user and policy creation with access key generation
- `budget-user-permissions.py` - updated IAM policy create/update script
- `main.tf` - Terraform resource definitions for IAM user, policy, access key, and budget
